What is an "artist mesh"?

Artist mesh : 3D 작업자가 손으로 만든 mesh, edge flow가 깨끗한 quad 구조

  • 머리카락 결 · 옷 솔기처럼 표면에 자연스러운 "결"
Fig. 2 · 왼쪽 두 패널이 artist-made, 오른쪽 두 패널이 기계가 만든 mesh. 눈으로도 차이가 보인다.
Reference · Paper §1 · Fig. 2
Speaker Notes
  • "Mesh"는 3D 모델을 이루는 삼각형(또는 사각형) 덩어리 — 표면을 조각난 패널로 덮은 것
  • Artist mesh 특징: 사각형(quad) 위주 + edge가 표면 곡률 방향을 따라 흐름
  • 비유: 옷의 솔기가 어깨·허리 같은 자연스러운 선을 따라 배치되는 것과 같음
  • 왜 중요? 리깅/애니메이션, 텍스처링, 시뮬레이션이 모두 edge flow 위에서 돌아감
  • CLO 맥락: 가먼트 mesh도 결국 이 "결"이 있어야 시뮬레이션이 안정적 — 다음 슬라이드에서 "왜 AI가 이걸 못 만드는지"

Why can't generative AI make this?

요즘 3D 생성 AI (TRELLIS · Hunyuan3D · CLAY · Wonder3D) : Marching Cubes 같은 알고리즘으로 reconstruction

  • dense triangle 덩어리
  • 그래서 AR transformer로 mesh 직접 생성하는 새 흐름 — 다음 세 논문이 대표
Marching Cubes 출력 예시 (AI 아닌 단순 알고리즘 결과) — 표면은 복원되지만 triangle이 uniform · dense
Reference · Paper §1 "[Wonder3D · TRELLIS · CLAY · Hunyuan3D-2.5 ...] typically results in dense triangle meshes..."
Speaker Notes
  • 대부분의 3D 생성 AI는 implicit representation (SDF, occupancy field) → 표면 추출 단계에서 Marching Cubes
  • MC는 voxel grid를 삼각형으로 균등하게 쪼갬 — 표면 방향·curvature 무시
  • 결과: 10–100만 face 짜리 "삼각형 수프" (triangle soup)
  • 외형(렌더 이미지)은 OK지만 실제 production 파이프라인엔 못 씀 — 리깅/애니메이션/시뮬레이션 불가능
  • 이게 3D 생성 AI의 숨겨진 한계 — 관객이 평소 보는 데모는 "뽀샵된 썸네일"

MeshGPT (2024) — the origin

MeshGPT : triangle마다 VQ-VAE discrete token → GPT 시퀀스 예측

  • AR mesh 생성 계보의 원조
MeshGPT Fig. 1 (Siddiqui et al. CVPR 2024)
방식

각 triangle을 VQ-VAE codebook embedding에 매핑, causal transformer가 다음 triangle 토큰을 예측. Mesh 생성을 sequence modeling으로 푼 최초 framework.

한계

Face 수 상한이 약 800으로 복잡한 production asset은 불가. Codebook이 triangle 형태 variation을 모두 담아야 해서 scale이 어려움.

Token example · MeshGPT (RVQ, 6 codes / face)
face₁  v₁ → c₃₂ c₁₈ v₂ → c₄₇ c₆ v₃ → c₂₅ c₃₉
face₂  v₁ → c₈ c₄₁ v₂ → c₃₂ c₁₉ v₃ → c₁₃ c₅₅
AR sequence → c₃₂ c₁₈ c₄₇ c₆ c₂₅ c₃₉ │ c₈ c₄₁ c₃₂ c₁₉ c₁₃ c₅₅ │ …  (face당 6 토큰, 총 6N)
Reference · Siddiqui et al. CVPR 2024 · SATO §2.2.1
Speaker Notes
  • 2024 CVPR — mesh를 sequence로 보는 발상의 시작
  • Triangle마다 512-dim embedding을 9개 codebook token으로 분해 후 causal transformer 예측
  • SATO도 "early representative"로 본문에 명시
  • 핵심 한계: face 수 ~800 상한 — 복잡한 production asset은 불가
  • VQ-VAE가 "모든 triangle 모양"을 학습해야 하는 부담 — scale이 본질적으로 어려움

BPT (2025) — push the scale

BPT : block (좌표 압축) + patchified (hub vertex fan으로 face 묶기) → 시퀀스 ~75% 단축

  • 더 큰 asset까지 AR 생성 확장
  • 단, patch 경계마다 edge flow 리셋
Fig. 1 · BPT로 생성한 mesh (최대 8k faces)
Fig. 3 · (a) Cartesian → block-wise 인덱스  ·  (b) hub vertex 주변 face → patch
한계

Patch 경계마다 edge flow가 끊김 — artist의 긴 결이 복원 안 됨. Vocabulary가 40,960으로 폭증.

Token example · BPT (block + patchified)
patch₁  hub: b′₁₁ o₂₃ v₁: b₃ o₄₂ v₂: o₁₇ v₃: b₅ o₈
patch₂  hub: b′₂ o₅₁ v₄: b₇ o₃₉ v₅: o₅ …
b′ (dual-block) = 새 patch 시작 마커 · b/o = block index · offset · v₂는 v₁과 같은 block이라 b 생략 → vertex당 1~2 tokens
Reference · Weng et al. 2025 · SATO §2.2.1 · Table 1
Speaker Notes
  • 2025 — "Blocked and Patchified Tokenization"
  • MeshGPT 대비 시퀀스 크게 짧아져 더 큰 mesh 가능
  • 하지만 block/patch 경계마다 strip-like continuity 파괴
  • SATO가 "patch-based ordering의 대표 문제 사례"로 직접 비판
  • Vocabulary 40,960 — combinatorial patch variation 때문에 폭증

DeepMesh (2025) — tighten the prior

DeepMesh : hierarchical coord + DPO로 품질 향상

  • SATO가 prefix sharing 그대로 계승한 직계 전작
Fig. 1 · DeepMesh 생성 결과 (Zhao et al. 2025)

한계

여전히 patch-based local traversal. 긴 edge flow를 하나의 chain으로 담아내지 못해 artist topology 보존에 실패.

Token example · DeepMesh (hierarchical coord + prefix sharing)
v₁ 5 · 12 · 101 │ v₂ 88 │ v₃ 7 · 54 │ v₄ 9 · 3 · 22
(c₁, c₂, c₃) = 4³/8³/16³ 계층 셀 인덱스 · v₂는 c₁·c₂ 공유로 c₃만 (1 token) · v₃는 c₁ 공유로 c₂·c₃ (2 tokens) · v₄는 c₁ 달라져 full (3 tokens) → vertex당 1~3 tokens
Reference · Zhao et al. 2025 · SATO §2.2.1 · §4.1
Speaker Notes
  • 2025 — hierarchical coord tokenization 도입
  • (c₁, c₂, c₃) coarse-to-fine + prefix sharing으로 시퀀스 길이 대폭 축소
  • DPO로 human preference까지 반영 — "artist가 선호하는 mesh"에 근접
  • SATO는 이 tokenization 철학을 그대로 계승하고 ordering만 strip으로 교체
  • 한계: 여전히 patch 단위 traversal이라 prefix sharing이 자주 리셋됨

Key insight — Triangle strips

Strip : 새 face가 이전 face와 edge 공유하며 자라는 구조

  • artist가 mesh를 incremental하게 grow하는 방식과 정렬
Triangle strip — 인접 face가 edge를 공유하는 ordered chain
Reference · Paper §3.1 Triangle Strips
Speaker Notes
  • 고전 CG (1990년대~)에서 rendering 최적화 위해 쓰던 개념 — GPU primitive
  • 각 새 face가 단 하나의 new vertex만 추가 → memory-efficient
  • 중요한 건: connectivity + local continuity를 시퀀스 순서 자체에 인코딩한다는 점
  • Artist가 boundary에서 면을 incremental하게 확장하는 방식 = strip-like
  • SATO의 기여: 이 classic idea를 AR token의 단위로 승격 — tokenization의 mid-level language

Pipeline overview

Pipeline : point cloud → encoder → hourglass transformer → stride σ

Hierarchical geometry quantization

Quantization : 512³ voxel → 4³/8³/16³ 3-level, 상위 prefix 공유

  • 대부분 정점이 1–2 토큰
Fig. 4 (left) · (c₁, c₂, c₃) 3-level hierarchical coordinates
Reference · Paper §4.1 · Fig. 4 (left)
Speaker Notes
  • vertex 좌표를 512³ voxel grid에 정렬
  • (c₁, c₂, c₃) = 4³ / 8³ / 16³ — coarse to fine
  • 연속 정점이 같은 상위 cell에 속하면 c₁·c₂ 생략, c₃만 기록
  • 실제 token 분포: c₁ 20.7% / c₂ 35.0% / c₃ 44.3%
  • DeepMesh의 prefix sharing을 그대로 계승 — strip의 spatial continuity와 궁합 좋음
  • 비전문가용 비유: 주소를 "서울시 강남구 역삼동 123"이라고 매번 쓰지 않고, 구와 동이 같으면 번지만 — 그런 압축

Strip extraction — zipper-like growth

Strip extraction : lowest-coord seed → adjacency 따라 zipper처럼 grow → dead-end에서 끊기

  • deterministic해서 모델이 학습하기 쉬움
Reference · Paper §4.2.1 · Algorithm 1
Speaker Notes
  • Edge-to-face adjacency map 먼저 구축, visited[f]=false
  • Seed는 lowest-coordinate-first — spatially coherent pattern 유지
  • δ=1이면 1 vertex 추가 (triangle), δ=2이면 2 vertex 추가 (quad)
  • Dead-end (boundary 또는 visited만 남음) 도달 시 strip 종료, C₁ʹ 토큰 emit
  • Greedy decomposition이 최적은 아니지만, "일관된 traversal"이 학습 가능성에 더 중요하다는 게 SATO의 철학

Native UV segmentation via C₁ʺ

Native UV seg : 같은 시퀀스 안에서 UV island partition

  • C₁ʺ 토큰이 island 경계 표시
Fig. 5 · UV part 단위 traversal — SATO가 학습하는 partition 구조
① Island 1 faces
strip → c₁c₂c₃ c₃ c₃ a* c₁c₂c₃ c₃
↓ C₁ʺ island 경계 (GT에서 미리 정의)
② Island 2 faces
strip → c₁c₂c₃ c₃ c₂c₃
↓ C₁ʺ island 경계
③ Island 3 faces
strip → c₁c₂c₃ c₃ …
주의  (u,v) 좌표 자체는 미생성 — face→island 분할만. 평면화는 별도 parameterizer 필요.
Reference · Paper §4.2.3 · Fig. 5
Speaker Notes
  • Mesh face를 UV island로 disjoint partition
  • Island traversal 순서는 bottom-to-top deterministic
  • Island 내부는 strip extraction 그대로, 단 다음 seed는 현재 island 안에서만
  • C₁ʺ 토큰은 C₁ʹ 기능 포함 — strip 종료 + UV 전환 동시 표시
  • AR mesh 생성 framework에서 UV 분할까지 한 번에 내는 건 이게 최초
  • (u, v) 자체가 안 나온다는 점은 Q&A에서도 자주 나오는 포인트 — 명확히 짚고 넘어가자

Unified tri/quad via stride σ

Unified tri/quad : 같은 토큰 시퀀스, σ 스위치만

  • σ=1 → triangle · σ=2 → quad, 재학습 불필요
Fig. 6 · (a) tri strip은 turn 가능 (b) quad strip은 직진 (c) quad 토큰을 tri로도 디코딩 가능
Reference · Paper §4.2.1 · §4.3 · Fig. 6
Speaker Notes
  • σ=1: 각 새 vertex가 fᵢ = (vᵢ, vᵢ₊₁, vᵢ₊₂) 완성
  • σ=2: vertex pair로 qᵢ = (v₂ᵢ, v₂ᵢ₊₁, v₂ᵢ₊₃, v₂ᵢ₊₂) 완성
  • Winding 일관성 위해 face 생성 시 마지막 두 vertex swap → tri/quad 시퀀스 identical
  • Quad strip은 edge flip turn이 거의 없어 prior가 깨끗 → tri 품질까지 끌어올림
  • 고품질 quad mesh 120개만으로 fine-tune 가능 (tri prior가 이미 strong하니까)
